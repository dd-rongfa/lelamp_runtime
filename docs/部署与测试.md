# 小灯（LeLamp 豆包版）部署与测试指南

面向想 clone 下来跑一跑的人。**不需要树莓派、不需要机械臂**，一台带麦克风的电脑就能体验
小灯的「语音对话 + 看图 + 调用工具」。有树莓派 / LeLamp 真机的，往下看进阶部分。

> 架构原理见 [`ARCHITECTURE.md`](./ARCHITECTURE.md)。这里只讲怎么跑、怎么验、怎么排坑。

---

## 0. 你能测到什么

| 你有什么 | 能测的能力 | 难度 |
|---|---|---|
| **只有电脑**（Windows / macOS / Ubuntu） | 语音对话 + 看图 + 工具调用（机械臂/灯走 mock，只打日志） | ⭐ 简单 |
| **树莓派**（没买真机） | 同上，在 Pi 上跑软件链；可接 USB 摄像头/麦 | ⭐⭐ |
| **LeLamp 真机** | 机械臂真动、灯真亮 + 上面全部 | ⭐⭐⭐ 要装硬件依赖 + 标定 |

> ⚠️ 真机硬件路径（机械臂/灯）目前**未经充分验证**，遇到问题请反馈。软件链（语音/视觉/工具）已实测可跑。

**支持平台（软件链）**：Windows、macOS（Intel/Apple 芯片）、Ubuntu x86_64、树莓派 **64 位** 系统——
关键原生库（livekit/onnxruntime/opencv/numpy）这些平台都有预编译 wheel，`uv sync` 直接装。
⚠️ 树莓派**必须 64 位系统**（Pi OS 64-bit / Bookworm 或 Bullseye）；32 位（armhf）没有 wheel，装不上。
Python 3.12 由 uv 自动装好（Pi OS 自带 3.11 不用管）。

## 1. 准备：火山豆包凭据（必须，自己的）

小灯的大脑是租来的（火山豆包），你需要**自己的**两把 key（同一个火山账号下）：

| 用途 | 去哪拿 | 填到 .env 的 |
|---|---|---|
| 语音 STT/TTS | 火山引擎控制台 → 语音技术 → 大模型语音，开通后拿「新版 v3 单 X-Api-Key」 | `VOLCENGINE_VOICE_API_KEY` |
| LLM + 视觉 | 火山引擎控制台 → 方舟(Ark) → API Key | `LLM_API_KEY` |

> 默认模型是全模态 `Doubao-Seed-2.0-lite`（聊天 + 看图一颗脑）。去方舟控制台**开通**这个模型的使用权。
> 费用很低，按调用计费；测试花不了几块钱。

## 2. 无硬件快速开始（任何电脑）

```bash
# 1) 装 uv：https://docs.astral.sh/uv/getting-started/installation/

# 2) 拉代码 + 装依赖（只装语音那套轻依赖，不碰硬件库）
git clone https://github.com/dd-rongfa/lelamp_runtime.git
cd lelamp_runtime
uv sync

# 3) 配凭据
cp .env.example .env          # Windows PowerShell: copy .env.example .env
#   编辑 .env，填 VOLCENGINE_VOICE_API_KEY 和 LLM_API_KEY 两把 key
#   .env 里已默认 LELAMP_NO_HARDWARE=1（机械臂/灯走 mock）

# 4) 先验证 key 通不通（不用麦克风）
uv run tools/smoke_doubao.py

# 5) 接麦克风对话
uv run main.py console
```

**Ubuntu 额外一步**：跑 `console`（接麦）前先装音频库
```bash
sudo apt install libportaudio2
```

## 3. 怎么验证它在工作

### 3a. 不开麦：三段式冒烟（最先跑这个）

```bash
uv run tools/smoke_doubao.py            # 跑全部三段
uv run tools/smoke_doubao.py --stage llm   # 只测某段：llm / tts / stt
```

逐段打印 `PASS / FAIL / SKIP` 和延迟。全 PASS 就说明你的 key 和网络都通了。
参考延迟（境内）：LLM 首字 ~0.7-1.5s、TTS 首帧 ~0.7s、STT 首结果 ~0.6-0.9s。

### 3b. 开麦：跟小灯对话

`uv run main.py console` 起来后，直接说话。试试这几句看三大能力：

| 你说 | 预期 |
|---|---|
| 「你好呀」 | 用中文搭话（mock 模式下动作/灯光只在日志里） |
| 「**你看看这是啥？**」 | 调 `look` 看 `assets/images/1_lamp_3d.png`（仓库自带）并描述 |
| 「**开心地点个头**」 | 调 `play_recording`（真机才会动，mock 打日志） |
| 「**把灯变成红色**」 | 调 `set_rgb_solid`（同上） |

## 4. 让小灯看实时画面（摄像头）

**默认不装摄像头库**，`look` 会退回看仓库自带的测试图（并会如实说明"这是测试图、不是实景"）。
想让它看你眼前的真实画面（笔记本摄像头 / USB 摄像头都行）：

```bash
uv sync --extra vision        # 装 opencv（轻量，不拉机械臂那些重依赖）
uv run main.py console
```

装了之后**摄像头自动优先**——有摄像头就看实景，没有才退回测试图，`.env` 里的
`LAMP_VISION_IMAGE` 不用动。多摄像头可用 `LAMP_CAMERA_INDEX` 选序号。
说「你看看我」试试。

## 5. 真机：树莓派 + LeLamp 机械臂

让机械臂真动、灯真亮，比纯软件多几步，且 ARM 上装依赖较慢，留足时间。

```bash
# 系统库
sudo apt install portaudio19-dev

# 装硬件依赖（lerobot/torch/rpi-ws281x 等，ARM 上慢，是最容易出问题的一步）
uv sync --extra hardware

# .env 里去掉 LELAMP_NO_HARDWARE（或设 0）

# 舵机：找口 → 设 ID → 标定（首次必做）
uv run lerobot-find-port
uv run -m lelamp.setup_motors --id lelamp --port <上一步的口>
sudo uv run -m lelamp.calibrate --id lelamp --port <同上>

# 硬件自检
sudo uv run -m lelamp.test.test_rgb
uv run -m lelamp.test.test_audio

# 跑（GPIO 要 root）
sudo uv run main.py console
```

> 开机自启（systemd）见 [README 的「Start upon boot」](../README.md#4-start-upon-boot)。

## 6. 常见问题排查

| 现象 | 原因 / 解法 |
|---|---|
| smoke 的 LLM 段 `401 / AuthenticationError` | `LLM_API_KEY` 错或没开通方舟该模型 |
| smoke 的 TTS 段 `55000000 resource ID mismatched` | 音色和 key 不匹配；用默认 `LAMP_SPEAKER=zh_female_vv_uranus_bigtts` |
| 回复要等 7~12 秒才出声 | 豆包 Seed 默认开了「思考」；`.env` 设 `LLM_THINKING=disabled`（默认已关，别手动开） |
| `console` 报找不到音频设备 / 没声音 | 没接麦克风/扬声器；Ubuntu 先 `apt install libportaudio2`，检查系统默认输入输出 |
| `look` 说「看不见」 | 没配 `LAMP_VISION_IMAGE` 又没摄像头；用 .env.example 默认那行即可 |
| `uv sync --extra hardware` 在 Pi 上失败 | lerobot/torch 在 ARM 上编译/下载慢或缺 wheel；先确认网络，必要时分开装、看具体哪个包报错 |
| 旁边人说话也被搭理 | 设计就是常开免唤醒、没有唤醒词，属正常 |

## 7. 反馈

测试时请记录：你的系统（Windows/Ubuntu/Pi 型号）、卡在哪一步、完整报错。
smoke 的 PASS/FAIL + 延迟数也一并发来，方便定位是 key、网络还是代码问题。
